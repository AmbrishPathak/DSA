DSA & Algorithms Cheatsheet

Your one-page guide for cracking the tech interview.

Arrays & Strings

Key Concept

Stores elements in contiguous memory blocks. O(1) access with index. Strings are immutable arrays of characters.

Common Use Cases

  • Storing collections of items.
  • Sliding Window, Two Pointers patterns.
  • Implementing other data structures.

Python Snippet: Two Pointers

def find_pair(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return (left, right)
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    return None

Hash Tables & Sets

Key Concept

Key-value pairs (or just keys for sets) stored based on a hash function. Provides O(1) average time complexity for insert, delete, and search.

Common Use Cases

  • Caching (e.g., memoization).
  • Counting frequencies of items.
  • Detecting duplicates or finding intersections.

Python Snippet: Frequency Count

from collections import Counter
# In Python, dictionaries are hash tables
my_list = ['a', 'b', 'a', 'c', 'b', 'a']
freq_map = {}
for item in my_list:
    freq_map[item] = freq_map.get(item, 0) + 1

# Or simply use Counter
freq_counter = Counter(my_list)
# freq_counter is {'a': 3, 'b': 2, 'c': 1}

Stacks & Queues

Key Concept

Stack (LIFO): Last-In, First-Out. Operations are push and pop. Queue (FIFO): First-In, First-Out. Operations are enqueue and dequeue.

Common Use Cases

  • Stack: DFS, parentheses validation, call stack.
  • Queue: BFS, task scheduling, message processing.

Python Snippet: Queue with deque

from collections import deque

queue = deque()

# Enqueue
queue.append('Task 1')
queue.append('Task 2')

# Dequeue
processed_task = queue.popleft() # 'Task 1'

# Python lists can be used as a stack
stack = []
stack.append(1) # Push
stack.pop()     # Pop

Recursion & Backtracking

Key Concept

A function that calls itself to solve smaller instances of the same problem. Backtracking explores all potential solutions, abandoning a path if it's invalid.

Common Use Cases

  • Tree and graph traversals (DFS).
  • Generating permutations/combinations.
  • Solving mazes, Sudoku, N-Queens.

Python Snippet: Backtracking Template

def backtrack(candidate, input_data):
    if is_solution(candidate):
        add_to_results(candidate)
        return

    for next_candidate in generate_candidates(candidate):
        # Choose
        add_to_candidate(next_candidate)
        # Explore
        backtrack(candidate, input_data)
        # Unchoose (backtrack)
        remove_from_candidate(next_candidate)

Trees (BT & BST)

Key Concept

Hierarchical data structure with a root node and child nodes. A Binary Search Tree (BST) is ordered: left child < parent < right child, enabling O(log n) search.

Common Use Cases

  • Representing hierarchy (file systems, DOM).
  • BSTs for efficient searching and sorting.
  • Tries for autocompletion.

Python Snippet: In-order Traversal

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def in_order_traversal(node):
    if not node:
        return
    in_order_traversal(node.left)
    print(node.val) # Visit node
    in_order_traversal(node.right)

Heaps / Priority Queue

Key Concept

A tree-based structure satisfying the heap property (parent is >= children for max-heap, or <= for min-heap). Efficiently finds the min/max element.

Common Use Cases

  • Finding the K-th largest/smallest element.
  • Dijkstra's and Prim's algorithms.
  • Task scheduling based on priority.

Python Snippet: heapq (Min-Heap)

import heapq

scores = [23, 10, 54, 12, 88]
# Python's heapq is a min-heap
heapq.heapify(scores) # O(n)

# Add a new score
heapq.heappush(scores, 45) # O(log n)

# Get the smallest score
smallest = heapq.heappop(scores) # 10, O(log n)

# To simulate a max-heap, store negated values

Graphs

Key Concept

A collection of nodes (vertices) connected by edges. Can be directed or undirected, weighted or unweighted. Traversed using BFS or DFS.

Common Use Cases

  • Social networks, web page links.
  • Mapping and navigation (Google Maps).
  • Dependency resolution (e.g., package managers).

Python Snippet: Adjacency List

from collections import defaultdict

# Graph of user connections
graph = defaultdict(list)
edges = [('A', 'B'), ('B', 'C'), ('A', 'C')]

for u, v in edges:
    graph[u].append(v)
    graph[v].append(u) # For undirected graph

# graph['A'] is ['B', 'C']

Sorting

Key Concept

Arranging elements in a specific order (e.g., ascending). Key algorithms: Merge Sort, Quick Sort, Heap Sort (all O(n log n) average).

Common Use Cases

  • Fundamental for data processing.
  • Prerequisite for binary search.
  • Identifying patterns or outliers in data.

Python Snippet: Built-in Sort

numbers = [3, 1, 4, 1, 5, 9, 2]

# In-place sort (Timsort, O(n log n))
numbers.sort() 
# numbers is now [1, 1, 2, 3, 4, 5, 9]

# Sort with a custom key (e.g., by length of string)
words = ['banana', 'pie', 'apple']
words.sort(key=len)
# words is now ['pie', 'apple', 'banana']

Binary Search

Key Concept

A "divide and conquer" search algorithm with O(log n) time complexity. It repeatedly divides the search interval in half. Requires a sorted array.

Common Use Cases

  • Efficiently searching in large, sorted datasets.
  • Finding the position to insert an element.
  • Used in problems that can be mapped to a search space.

Python Snippet: Iterative Search

def binary_search(arr, target):
    low, high = 0, len(arr) - 1
    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1 # Not found
